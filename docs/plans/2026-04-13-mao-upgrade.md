# 前置调研器

> "没有调查，就没有发言权。"
> —— 毛泽东，《反对本本主义》（1930）

> "感觉到了的东西，我们不能立刻理解它；只有理解了的东西，才更深刻地感觉它。"
> —— 毛泽东，《实践论》（1937）

## 核心任务

**在开始解剖或建设之前，先做两件事：建立感性认识基础；判断所处阶段。**

不做这两件事的代价是：
- **解剖**变成"用训练数据里的印象攻击目标"——证据是幻觉，结论是废话
- **建设**变成"从理论推导解法"——方案在真空里成立，一碰现实就崩

## 模块 A：感性认识建立（调查研究）

> 来源：群众路线——"从群众中来，到群众中去"；"一切结论产生于调查情况的末尾，而不是在它的先头"

### 操作步骤

**Step A1：识别需要调查的核心问题**

用一句话写出：我需要了解什么才能开始解剖/建设？

格式：`为了[解剖/建设目标]，我需要先搞清楚[核心问题]`

**Step A2：确定 3-5 个一手来源**

一手来源的优先级（高→低）：
1. 原始文档、官方规格、原始论文、实测数据
2. 实践者的具体案例描述（非营销性）
3. 同行评审或独立评测
4. 综合报告（需标注来源机构的立场）

**禁止使用的来源：**
- 营销材料、厂商白皮书（除非你同时有独立验证）
- 二手转述（"有人说"型来源）
- 没有具体数据的泛化叙述

**Step A3：提取关键事实**

对每个来源，只提取三类东西：
- **可量化的事实**（数字、时间范围、规模、成功/失败的具体条件）
- **边界条件**（在什么情况下这个来源的描述成立）
- **内部矛盾**（来源本身说法前后不一的地方）

**Step A4：感性认识小结**

```
【调查小结】
核心问题：[一句话]
已确认的事实（3-5条，每条附来源）：
  1. [事实] — 来源：[X]
  2. [事实] — 来源：[X]
  ...
仍不确定的关键点：
  - [不确定点1]
  - [不确定点2]
感性认识是否足够进入下一步？[是/否，若否说明还需要什么]
```

## 模块 B：阶段判断（论持久战三阶段论）

> 来源：毛泽东《论持久战》（1938）
> "战争的过程究竟会要怎么样？……不是每个人都解决了的。"

《论持久战》的核心方法论不是关于"战争"的，而是关于**在变化中识别阶段，并在每个阶段采用与之匹配的主要矛盾和战略方针**。

三个阶段的结构：

| 阶段 | 特征 | 主要矛盾方向 | 战略重心 |
|------|------|------------|---------|
| **战略防御期**（探索期） | 对方强、我方弱；方向未定；首要任务是生存和学习 | 我方能否在压力下存活并积累知识？ | 运动战：不死守、快速学习、避免被一口吃掉 |
| **战略相持期**（扩张期） | 力量基本对等或开始积累；方向已定；首要任务是建立优势 | 如何将局部优势转化为全局优势？ | 游击战为主：大量积累实践经验，消耗对方资源 |
| **战略反攻期**（存量期） | 我方已有规模优势；首要任务是巩固和精细化 | 如何防止优势因管理复杂度上升而流失？ | 阵地战：系统化、标准化、精细运营 |

**注：** 这三个阶段名称在技术/产品场景下替换为（探索期/扩张期/存量期），机制相同。

### 操作步骤

**Step B1：识别当前阶段**

回答以下诊断问题：

```
诊断问题（每项选 A/B）：

1. 对这个问题/领域/市场，我方是否已有成熟方向？
   A. 否，方向仍在探索     → 倾向探索期
   B. 是，方向已定，在执行 → 倾向扩张期或存量期

2. 当前的主要挑战是什么？
   A. 不知道做什么才对      → 探索期
   B. 知道做什么，但做不够快 → 扩张期
   C. 规模有了，但效率/质量下滑 → 存量期

3. 资源（时间/人力/资金）相对于目标来说：
   A. 极度紧张，需要精打细算  → 探索期
   B. 足够支撑快速扩张       → 扩张期
   C. 充足但面临复杂度压力   → 存量期
```

**Step B2：确定阶段匹配的主要矛盾**

| 阶段 | 主要矛盾一句话 |
|------|-------------|
| 探索期 | "已知信息"与"决策所需信息"之间的矛盾——信息不足导致无法做出有效决策 |
| 扩张期 | "执行速度"与"质量保障"之间的矛盾——快与好的张力决定扩张的可持续性 |
| 存量期 | "规模优势"与"管理复杂度"之间的矛盾——规模越大，熵增压力越大 |

**Step B3：确定阶段匹配的战略方针**

| 阶段 | 战略方针 | 解剖重点 | 建设重点 |
|------|---------|---------|---------|
| 探索期 | 快速学习，保持灵活，不做重大承诺 | 重点扫描：约束层（边界在哪里）、历史层（有没有失败先例） | 小实验优先，快速验证，不求完整 |
| 扩张期 | 建立系统，固化优势，消耗对方资源 | 重点扫描：机制层（因果链是否成立）、动机层（谁在阻力方向） | 标准化流程，培养团队能力，加速复制 |
| 存量期 | 精细化，防守，优化内部矛盾 | 重点扫描：逻辑层（现有方案的内部一致性）、工程层（可维护性） | 技术债清理，监控告警，防止退化 |

**Step B4：阶段判断输出**

```
【阶段判断报告】
当前阶段：[探索期 / 扩张期 / 存量期]
判断依据：[2-3句]
主要矛盾：[阶段匹配的一句话]
战略方针：[一句话]
对后续解剖/建设的影响：
  - 解剖时应重点扫描：[维度]
  - 建设时应重点关注：[方向]
  - 本阶段不适合做的事：[列举1-2条，避免"阵地战用在探索期"类错误]
```

# 正向实践规划器

> "从群众中集中起来，又到群众中坚持下去，以形成正确的领导意见，这是基本的领导方法。"
> —— 毛泽东，《关于领导方法的若干问题》（1943）

> "战略上的持久战，战役和战斗上的速决战。"
> —— 毛泽东，《论持久战》（1938）

## 核心任务

**从真实场景出发，生成阶段匹配的、可操作的实践规划。**

这个 skill 解决的问题：
1. 方案从理论推导而来，不接地气——用群众路线调查驱动
2. 方案不考虑当前阶段，把扩张期战术用在探索期——用三阶段论校准
3. 方案太宏大，无法落地——强制输出"最小有效实践"和"第一步具体动作"

**前置条件：** 建议先运行 `aolun-ground` 完成感性认识建立和阶段判断。如果没有 ground 报告，本 skill 的第一步强制内嵌一个简化版调查。

## 第一层：调查先于方案

> "凡是没有把握的事情，就要作调查研究，与实际情况相符合的才有把握。"

### 1.1 确认是否已有 ground 报告

- **有 ground 报告**：直接使用，跳到第二层
- **没有 ground 报告**：执行以下简化调查

**简化调查（5分钟版本）：**

```
目标用户/受益方是谁？（具体的人，不是"用户"）
他们目前用什么方法解决这个问题？（现有做法，不是理想做法）
现有方法最大的痛点是什么？（他们自己说的，不是你推测的）
你是从哪里知道上面这些的？（来源）
```

**禁止跳过：** 如果上述四个问题有任何一个回答是"我觉得""应该""大概"，必须先做调查，不得继续。

### 1.2 识别真实需求 vs 表达需求

群众路线的核心洞察：人们表达的需求（want）往往不等于真实需求（need）。

| 类型 | 特征 | 处理方式 |
|------|------|---------|
| 表达需求 | "我需要一个更快的系统" | 追问：快到什么程度？现在多慢？慢在哪里？ |
| 真实需求 | "用户等待超过3秒就放弃" | 直接作为规划输入 |
| 假性需求 | "我需要AI" | 追问：AI要解决什么具体问题？没有AI现在怎么做？ |

**输出：**
```
真实需求陈述：[一句话，包含可测量的具体条件]
来源：[怎么知道的]
```

## 第二层：识别主要矛盾

> 来源：《矛盾论》——"解决矛盾的方法，即解决矛盾的具体办法，将随矛盾的性质之不同而不同。"

### 2.1 从真实需求中提取矛盾

基于调查结果，找出阻碍真实需求实现的核心张力：

**格式：** `[现状A]与[目标B]之间的矛盾，当前[A]处于支配地位`

例：
- 探索期：`"我们对用户行为的了解"与"做出有效决策所需的了解"之间的矛盾，当前信息不足处于支配地位`
- 扩张期：`"功能交付速度"与"系统稳定性"之间的矛盾，当前速度需求处于支配地位`
- 存量期：`"系统功能数量"与"维护团队认知负载"之间的矛盾，当前复杂度处于支配地位`

### 2.2 阶段匹配验证

从 ground 报告读取当前阶段，验证识别的矛盾是否与阶段匹配：

| 阶段 | 矛盾是否匹配的检验问题 |
|------|---------------------|
| 探索期 | 矛盾是否聚焦于"获取有效信息"？ |
| 扩张期 | 矛盾是否聚焦于"建立可复制的优势"？ |
| 存量期 | 矛盾是否聚焦于"控制复杂度和维护质量"？ |

如果矛盾与阶段不匹配，说明阶段判断可能有误，或者矛盾识别不够准确，需要重新分析。

## 第三层：规划最小有效实践

> 来源：《实践论》——"真理的标准只能是社会的实践。"
> 论持久战策略——"战略上的持久战，战役和战斗上的速决战"

**核心原则：** 最小有效实践（Minimum Effective Practice, MEP）——能够产生真实反馈的最小规模的行动。不是"MVP"（最小可行产品），而是**能产生真实认识的最小实践单元**。

### 3.1 设计最小有效实践

**MEP 设计三原则：**

1. **小**：可以在当前阶段的资源约束内完成（探索期：1-2周；扩张期：1-2个月；存量期：视情况）
2. **真**：接触真实的用户/数据/环境，不是模拟
3. **有反馈**：执行完之后，你对主要矛盾的理解会明确变化（要么验证，要么推翻）

**MEP 设计模板：**

```
实践名称：[一句话描述做什么]
规模：[涉及多少用户/数据/团队]
时间：[多长时间]
执行的具体动作（第一步是什么）：[一句话，足够具体到明天可以开始]
预期产生的信息：[执行完之后，你会知道什么]
成功标准：[如何判断这个实践是否有效——具体的可测量条件]
失败/无效标准：[什么情况下这个实践没有产生有效信息]
```

### 3.2 阶段匹配的实践形态

| 阶段 | 对应战术 | 实践形态 |
|------|---------|---------|
| 探索期 | 运动战：快速移动，不死守 | 快速实验：A/B测试、用户访谈、原型验证；每次实践1-2周，产出具体数据 |
| 扩张期 | 游击战：积累优势，消耗对方资源 | 系统化实践：标准化流程、培训体系、可复制模板；每次实践1-2个月，产出可复制方法 |
| 存量期 | 阵地战：精细防守，优化内部 | 优化实践：性能优化、技术债清理、监控完善；以季度为单位，产出可量化的改善指标 |

## 第四层：再认识循环设计

> 来源：《实践论》——"实践、认识、再实践、再认识，这种形式，循环往复以至无穷。"

任何实践规划都必须内嵌一个"再认识"机制，否则实践执行完了，认识却没有更新。

### 4.1 设计反馈节点

```

### 第五步：普遍性边界检验

> 来源：《矛盾论》——"矛盾的普遍性和矛盾的特殊性的关系，就是矛盾的共性和个性的关系。"
> 
> 技术论断最常见的概念错误之一：把**特殊条件下的成立**当作**普遍规律**推广。

**检验问题：**

1. **样本边界**：这个结论来自什么样本？
   - 作者/来源的样本有哪些已知的系统性偏差？（行业、规模、团队文化、时间段）
   - 样本是否代表目标受众？

2. **条件特殊性**：这个结论在什么特殊条件下成立？
   - 去除这些条件，结论还成立吗？
   - 作者是否把这些条件当作普遍前提，而没有明说？

3. **普遍化程度判断**：
   
   | 普遍化程度 | 特征 | 对结论的影响 |
   |----------|------|------------|
   | 合理 | 结论明确标注了适用范围 | 结论可参考 |
   | 过度普遍化 | 特殊条件下成立的结论，被当作所有情况的规律 | 概念陷阱，需要在攻击文中指出 |
   | 故意混淆 | 作者有利益动机推动过度普遍化 | 既是概念问题，也是利益问题，需要联合利益层处理 |

> 输出格式：
> ```
> 普遍性边界：
>   - 结论的实际适用范围：[具体条件]
>   - 被过度普遍化的内容（如有）：[具体描述]
>   - 普遍化程度判断：[合理 / 过度 / 故意混淆]
> ```
```
