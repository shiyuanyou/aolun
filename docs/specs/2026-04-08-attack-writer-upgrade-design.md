# attack-writer 升级设计文档

**日期：** 2026-04-08  
**目标：** 把 `attack-writer` 从"风格手册"升级为有方法论支撑的辩证法战斗文写作系统

---

## 背景与问题

现有 `attack-writer/SKILL.md` 是一个完整的李敖风格写作器，三件武器（征引、解构、反讽）+ 三种结构模板 + 语言规范 + 质量自检。

问题：skill 只规定了**怎么打**，没有规定**打哪里**和**为什么打这里**。三件武器是工具，但没有前置的"目标分析框架"告诉 LLM 应该把武器对准哪个点。

结果：LLM 倾向于套模板，选择最显眼的弱点而不是最根本的弱点，"立"的部分是外加建议而不是从目标内部矛盾中生长出来。

---

## 解决方案：三层分层重构

把整个 skill 重构为三层，每层有明确的理论来源：

- **第一层**：辩证诊断（《矛盾论》全部概念驱动）
- **第二层**：攻击执行（原三件武器，但由诊断结果驱动，而非套模板）
- **第三层**：实践检验（《实践论》驱动，分两轨）
- **第四章**：认识的螺旋（《实践论》，单独成章，强调不完美中前行）

---

## 第一层：辩证诊断（《矛盾论》）

**目的：** 开打前，把目标的内部矛盾结构摸透。这是整个写作流程的前提——不做诊断，就不知道把武器对准哪里。

### 1.1 主要矛盾定位

目标内部有多个矛盾存在时，必须找到主要矛盾——"其中必有一种是主要的，起着领导的、决定的作用"（《矛盾论》四）。

**操作：**
- 列出目标中存在的所有内在张力（至少3条）
- 判断哪一条"规定或影响着其他矛盾的存在和发展"
- 一句话陈述主要矛盾：`[A]与[B]之间的矛盾，其中[A/B]处于支配地位`

### 1.2 矛盾特殊性分析

"不同质的矛盾，只有用不同质的方法才能解决"（《矛盾论》三）。不同类型的矛盾对应不同的解构路线。

**矛盾类型分类（四种）：**

| 类型 | 特征 | 对应解构路线 |
|------|------|------------|
| 概念混乱型 | 关键词被偷换或混用 | 暴露定义跳跃，精确指出在哪句话里 |
| 机制断裂型 | 因果链中间某环节缺失或虚构 | 还原隐藏约束，证明前提不成立 |
| 利益遮蔽型 | 论证服务于未申明的利益 | 以子之矛攻子之盾，用对方说法推翻对方 |
| 历史错位型 | 把历史上已经失败的方案重新包装 | 引出荒谬结论，让历史重演显出可笑 |

**操作：** 判断主要矛盾属于哪种类型，选定对应的主攻路线。（一个目标可以有多种类型，但只选一条主路线，其他作为补充。）

### 1.3 主要方面 vs 次要方面

矛盾有两个方面，"事物的性质，主要地是由取得支配地位的矛盾的主要方面所规定的"（《矛盾论》四）。

**操作：**
- 确认主要矛盾的两个方面各是什么
- 判断哪个方面目前处于支配地位（即对方论证依赖哪个方面来成立）
- 攻击策略：打击支配地位的那个方面，使其转化为次要方面

### 1.4 同一性分析：转化条件

矛盾的同一性意味着"矛盾着的双方，依据一定的条件，各向着其相反的方面转化"（《矛盾论》五）。

**操作：**
- 找出主要矛盾发生转化需要什么条件
- 这个条件就是"立"的方向——不是外加建议，而是目标内部矛盾运动的自然出口
- 一句话陈述转化条件：`当[条件X]成立时，[A]将让位于[B]`

### 1.5 对抗性判断

"对抗是矛盾斗争的一种形式，而不是矛盾斗争的一切形式"（《矛盾论》六）。

**操作：** 判断这个矛盾是对抗性的还是非对抗性的。

- **对抗性矛盾**（利益根本对立，无法调和）：攻击要彻底，反讽要锋利，不留余地
- **非对抗性矛盾**（认知错误或利益可调和）：攻击要精准，建构方向要具体，给对方留改正空间

### 输出：矛盾定位（混合模式）

完成以上五步后，输出 2-3 句矛盾定位，呈现给读者：

> **矛盾定位：** [主要矛盾一句话] + [矛盾类型 + 主攻路线一句话] + [转化条件/立的方向一句话]

这 2-3 句出现在文章开头之前（作为写作框架的显式声明），不计入文章正文字数。

## 第二层：攻击执行（三件武器，由诊断驱动）

三件武器保留，但每件武器的**选择标准和执行路线**由第一层的诊断结果决定，不再套模板。

### 武器一：征引

**原有逻辑：** 每个主要指控有具体依据，对方原话优先。

**升级驱动：** 征引的选择不再是"找到一条证据"，而是"找到能暴露主要矛盾的证据"。

- 如果主要矛盾是概念混乱型：引用对方在不同语境下对同一概念的不同用法
- 如果主要矛盾是机制断裂型：引用对方断言因果关系但省略中间环节的原文
- 如果主要矛盾是利益遮蔽型：引用对方既声称客观又暴露立场的原文
- 如果主要矛盾是历史错位型：引用对方的核心主张 + 历史上相同主张的原始来源

### 武器二：解构

**原有技法：** 四种（以子之矛攻子之盾、暴露定义跳跃、还原隐藏约束、引出荒谬结论）。

**升级驱动：** 解构路线由 1.2 的矛盾类型决定——主路线对应矛盾类型，其余三种作为补充验证。

解构的节奏保持原有逻辑：从最小漏洞开始，逐步升级到核心弱点，让批评力度逐渐积累。

**新增约束：** 解构的终点不是"证明对方错了"，而是"暴露内部矛盾，让转化条件（1.4）显现"。解构结束的标志是：读者已经能看到，在什么条件下对方的主要方面会让位于次要方面。

### 武器三：反讽

**原有条件：** 在证据和逻辑充分之后，给对方一个优雅的定性。

**升级驱动：** 反讽的落点由 1.3 决定——反讽的是目前处于支配地位的那个方面，而不是对方整体。

- 对抗性矛盾（1.5判断为对抗性）：反讽可以更锋利，目标是让对方的支配地位在读者眼中彻底崩塌
- 非对抗性矛盾：反讽用"同情式挖苦"，指出荒谬但不封死改正空间

## 第三层：实践检验（《实践论》）

### 轨道 A：建构落地性检验

"不在于懂得了客观世界的规律性，因而能够解释世界，而在于拿了这种对于客观规律性的认识去能动地改造世界"（《实践论》）。

文章里的"立"——建构方向——必须满足：

- **有实践来源**：这个方向来自哪个领域的实践经验？不能是纯理论推导
- **有边界条件**：在什么情况下这个方向成立，在什么情况下不成立
- **有可操作性**：读者读完之后，能做的第一个具体动作是什么

检验方法：把"立"的部分单独提取出来，问：如果对方想按这个方向走，第一步做什么？如果答不上来，"立"就还是空话。

### 轨道 B：对手反击模拟

"实践、认识、再实践、再认识，这种形式，循环往复以至无穷"（《实践论》）。

文章写完后，模拟对手最可能的三种反击：

1. **最强反击**：如果对手的论证中有一个我没有充分处理的方面，是哪个？
2. **转移反击**：对手最可能把话题引向哪个方向来回避主要矛盾？
3. **情绪反击**：对手最可能用什么情绪化的说法来绕过逻辑？

对每种反击，给出一句预备回应（不必展开，只需在文章中预埋伏笔）。

## 第四章：认识的螺旋

**这一章的核心：不完美中前行。**

《实践论》明确反对两种错误：
- **右倾机会主义**："思想不能随变化了的客观情况而前进"——等到完美再发布，就是这个
- **左翼空谈主义**："思想超过客观过程的一定发展阶段"——过度引申、言过其实，也是错的

**对写作者的提醒：**

### 够用就发，不等完美

每篇攻击文都是当前认识阶段的产物。判断标准不是"有没有漏洞"（一定有），而是：
- 主要矛盾是否被准确定位？
- 最核心的弱点是否有充分依据？
- "立"的方向是否可操作？

三条满足，就可以发布。不满足的部分，留给下一轮。

### 发布即实践：对手反应是材料

发布之后，对手的反击、读者的疑问、新的事实出现，都是"再实践"的材料。下一轮写作时，用这些材料更新对目标的认识。

**具体操作：**
- 记录对手的实际反击（对比 轨道B 的预测，看预测是否准确）
- 如果出现没有预测到的反击，把它加入下一轮第一层诊断的矛盾清单
- 如果"立"的方向被证明不可操作，在下一轮更新转化条件（1.4）

### skill 本身也在迭代

这个 skill 是当前认识阶段的产物。每次使用后，如果发现某个步骤不够用或者产生了误导，应该更新 skill 本身。这个 skill 不是终点，是起点。

## 文章结构模板

保留原有三种结构，调整第一节措辞以适配"矛盾定位"输出：

### 结构一：标准攻击文（适合技术评论）

```
矛盾定位：[2-3句，写作框架声明，不计入正文]

第一节：问题的提出（50-100字）
第二节：概念解剖（200-400字）
第三节：机制与约束拆解（300-500字）
第四节：利益与动机（100-200字）
第五节：他山之石（200-400字）
第六节：结语（50-150字）
```

### 结构二：快速评论（适合推文/短帖回复）

```
矛盾定位：[内部使用，不输出到正文]
第一句：复述对方核心主张（5-20字）
中间：最致命弱点 + 具体证据（50-150字）
最后：一句有力量的定性（10-30字）
```

### 结构三：深度技术拆解（适合长文）

```
矛盾定位：[2-3句，作为引言的前置框架]
引言、解剖、弱点清单、历史前辈、推动者分析、更好解法、结论
```

```

结构类型：[标准攻击文 / 快速评论 / 深度拆解]

转化条件（立的方向）：[一句话]
对抗性：[对抗性 / 非对抗性]
攻击的主要弱点：[最多3条]
引入的跨领域解法：[来源领域 + 方案名]
建构可操作性：[读者的第一个具体动作]
文章字数：[约X字]
建议发布平台：[推文/技术博客/内部评审等]
```

## 质量自检

```
【第一层：辩证诊断】
- [ ] 主要矛盾已定位，不是表面现象
- [ ] 矛盾类型已确认，主攻路线已选定
- [ ] 主要方面已识别，攻击对准支配地位的那个方面
- [ ] 转化条件已找到，"立"从内部矛盾中生长而非外加
- [ ] 对抗性已判断，反讽力度与之匹配

【第二层：攻击执行】
- [ ] 每个主要指控有具体依据（来自解剖/扫描结论）
- [ ] 征引选择与矛盾类型匹配
- [ ] 解构路线对应主攻路线，不是套模板
- [ ] 解构终点已到达（读者能看到转化条件）
- [ ] 反讽在论证之后，落点在主要方面
- [ ] 没有用气势代替论证

【第三层：实践检验】
- [ ] "立"有实践来源，有边界条件，有可操作性
- [ ] 能回答：读者第一个具体动作是什么
- [ ] 三种对手反击已模拟，伏笔已预埋
- [ ] 没有人身攻击，批评的是论断和行为

【认识的螺旋】
- [ ] 三条发布标准满足（主要矛盾定位、核心弱点有据、立可操作）
- [ ] 对方优点（如果真有）被公平提及
- [ ] 字数与信息密度匹配，没有冗余铺垫
```
