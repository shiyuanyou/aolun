# fileflow 多源输入 + 全 skill 对齐设计

> 日期：2026-04-14  
> 修订：2026-04-16（v2 — 整合 review 反馈）  
> 状态：设计中

## 问题陈述

当前 `aolun-fileflow` 和相关 skill 有两类问题：

### P0：多源输入不支持

fileflow 只处理"用户粘贴全文 → 保存为 `00-original.md`"的场景。实际使用中常见：

- 用户提供一个文件路径（如 `/user/data/paper.md`）
- 用户提供一个目录（如 `/user/data/docs/` 下有多个 .md 文件，含 index.md）
- 目录下有索引文件关联多个子文件
- 用户的活文档系统（自有 index），需要反复分析，不应每次 cp

当前流程无法处理这四种输入。

### P1：全 skill 对齐缺失

`ed5fbf7` 重构了 fileflow 和 scan-orchestrator 为"主 agent 只管状态和调度，subagent 自己读文件"的架构，引入了：

- 输出文件必须包含 `【摘要】` 块
- 引用硬约束：必须附原文行号 + 直接引用，禁止转述
- 01-dissect-concept 后需更新 00-todolist.md 的语义线索

但以下 skill 未适配：

| Skill | 缺失 |
|-------|------|
| `aolun-dissect-concept` | 无 `【摘要】` 输出规范；无"更新 00-todolist.md 语义线索"指令 |
| `aolun-inter-dissect-mechanism` | 无 `【摘要】` 输出规范 |
| `aolun-inter-dissect-constraint` | 同上 |
| `aolun-inter-dissect-interest` | 同上 |
| `aolun-attack` | 无 `【摘要】` 输出规范 |
| `aolun-other-mountains` | 同上 |
| `aolun-scan-logic/history/engineering/motive` | 无引用硬约束（"必须附行号+直接引用"） |

### P1（新增）：路由断裂

aolun-arming 按输入字符数 ≥ 1500 路由到 fileflow。但文件路径字符串通常远少于 1500 字符，arming 会走短文本路径（意图问询），导致路径输入永远不触发 fileflow。

### P2（新增）：行号偏移

当前 `00-original.md` 格式在原文前插入 header 注释行，导致 subagent 报告的行号与原文行号不一致（偏移约 2 行）。

## 设计决策

### D1：多源输入——五形态策略

根据输入形态和用户意图自动选择策略：

| 输入形态 | 判断条件 | 存储模式 | 处理方式 |
|---------|---------|---------|---------|
| 粘贴文本 | 无路径，直接是内容 | 快照 | 保存为 `00-original.md`（现有逻辑不变） |
| 单文件路径（快照） | `.md` 文件，用户指定快照 | 快照 | cp 为 `00-original.md` |
| 单文件路径（引用） | `.md` 文件，用户指定引用 | 引用 | 不复制，`00-todolist.md` 记录源路径 |
| 目录路径（快照） | 目录路径，用户指定快照 | 快照 | cp 目录为 `00-sources/`，生成 `00-index.md` |
| 目录路径（引用） | 目录路径，用户指定引用 | 引用 | 不复制，生成 `00-index.md`（指向源目录） |
| 非 Markdown | `.pdf`/`.docx`/其他 | — | 提示用户转为 Markdown，退出流程 |

**快照 vs 引用的选择规则：**

默认行为——用户未明确指定时：

- 粘贴文本 → 快照（只有一份内容，无同步问题）
- 文件路径 → 询问用户（快照 or 引用?）
- 目录路径 → 询问用户（快照 or 引用?）

用户在活文档系统中工作时（文档会被反复修改和重新分析），应选择引用模式。

**快照模式：** cp 文件到任务目录，分析期间内容不会变（不怕源文件被修改/删除）。适合一次性分析外部帖子。

**引用模式：** 不复制文件，subagent 直接读取源路径。适合活文档迭代分析（游戏设计文档、项目文档等）。风险和缓解见 D2。

**非 Markdown 文件处理规则：**

遇到 `.pdf`、`.docx`、`.txt`（非纯文本）、或其他非 Markdown 格式的文件时，不尝试任何转换，直接输出：

> "当前仅支持 Markdown 格式（.md）的源文件。请将文件转为 Markdown 后重新提交。常见转换方式：pandoc、手动复制粘贴为 .md。"

然后终止 fileflow 流程。

目录场景下只识别 `.md` 文件，跳过其他格式。

### D1.1：异常处理

| 场景 | 检测方式 | 处理 |
|------|---------|------|
| 路径不存在 | 文件/目录检测 | 输出"文件/目录不存在：<路径>"，终止 |
| 目录为空或无 .md 文件 | ls + 过滤 | 输出"目录下未找到 Markdown 文件：<路径>"，终止 |
| cp 失败（权限等） | cp 命令返回值 | 输出错误信息，终止 |
| 目录下 .md 文件超过 50 个 | 文件计数 | 警告"检测到 <N> 个 Markdown 文件，分析可能耗时较长。是否继续？[继续] [选择子集]" |
| 引用模式下源文件在分析过程中被修改 | 00-todolist.md 记录启动时文件 hash | subagent 发现 hash 不匹配时警告；严重不一致时提示用户是否重启 |

### D2：行号偏移解决方案

**问题：** 当前 `00-original.md` 格式在原文前插入 header 注释行（2 行），导致 Read 工具报告的行号 = 原文行号 + 2。

**方案：** 所有模式的 `00-original.md` header 改为 HTML 注释，不占内容行号：

```markdown
<!-- aolun-original: 粘贴文本 | 共 342 行 | 快照 | 创建于 2026-04-16T10:30:00 -->

<用户提供的原始文本，从文件第1行开始>
```

- HTML 注释 `<!-- -->` 占 1 行，但 Read 工具对 HTML 注释的行号报告与内容行号一致（注释行 = 第 1 行，内容从第 2 行开始）
- **更优方案：** 注释行后空一行，原文从第 3 行开始。在引用格式规范中声明：`00-original.md 第 3 行起为原文内容，引用行号 = Read 报告行号 - 2`
- **最终方案：** 不插入任何 header。文件元信息统一记录在 `00-todolist.md` 的原文索引中。`00-original.md` 纯粹保存原文，第 1 行就是原文第 1 行。溯源信息（源路径、模式、时间戳）记录在 `00-todolist.md`

**目录模式不受此影响**——`00-sources/` 下保留原始文件不变。

### D3：快照 vs 引用模式详解

#### 快照模式（cp）

```
用户提供路径 → 选择快照模式 → cp 文件/目录到任务目录 → 后续 subagent 读写任务目录内的文件
```

- 单文件快照：`cp <源文件> <任务目录>/00-original.md`
- 目录快照：`cp -r <源目录> <任务目录>/00-sources/`
- 分析期间源文件变更不影响已复制的版本
- 适合一次性分析

#### 引用模式（不复制）

```
用户提供路径 → 选择引用模式 → 不复制，00-todolist.md 和 00-index.md 记录源路径 → subagent 直接读取源路径
```

- 单文件引用：`00-todolist.md` 记录源文件绝对路径，subagent 读取该路径
- 目录引用：`00-index.md` 扫描源目录生成索引，subagent 读取源目录下的文件
- 源文件变更会反映在读取中（活文档场景的预期行为）
- 引用模式下 `00-todolist.md` 记录启动时各源文件的 MD5 hash，subagent 读取时比对，不匹配则警告

**对于目录引用模式，如果用户的文档系统已有 index 文件：**

```markdown
## 原文索引
- 输入类型：引用模式（目录）
- 源目录：/path/to/game-design/
- 外部索引：/path/to/game-design/index.md  ← 用户提供，跳过自动生成
- aolun 索引：<任务目录>/00-index.md       ← 仍会生成结构索引（标题/公式/表格/代码块位置）
- 索引与状态：<任务目录>/00-todolist.md
```

用户可以选择让 aolun 直接使用现有 index（跳过自动生成），或让 aolun 基于 00-index.md 的模板重新生成结构索引。

### D4：00-index.md 生成方式

**快照模式（目录）：** `cp -r` 完成后，主 agent 在 fileflow Part 1 阶段使用以下工具组合生成索引：

```
1. bash: ls <任务目录>/00-sources/*.md        → 文件列表
2. bash: wc -l <任务目录>/00-sources/*.md      → 各文件行数 + 总行数
3. grep -n '^#+ ' <任务目录>/00-sources/*.md   → 章节标题索引
4. grep -n '^\|' <任务目录>/00-sources/*.md    → 表格行号（或搜索 '表\d'）
5. grep -n '```' <任务目录>/00-sources/*.md    → 代码块行号
6. grep -n '\$\$' <任务目录>/00-sources/*.md    → 数学公式块行号
7. 对每个文件 Read 前 3-5 行，提取摘要（前50字符）
```

**引用模式（目录）：** 来源路径替换为源目录绝对路径，扫描逻辑相同。

**引用模式 + 外部 index：** 文件清单部分直接引用外部 index，但结构索引（标题/公式/表格/代码块位置标注）仍由 aolun 生成（格式统一，方便 subagent 搜索）。

`00-index.md` 格式：

```markdown
# 原文索引 — <brief>

> 源目录：00-sources/                         ← 快照模式
> 或源目录：<源目录绝对路径>                     ← 引用模式
> 文件数：<N> 个 .md 文件
> 总行数：<M>

## 文件清单

| # | 文件路径 | 行数 | 摘要（前50字符） |
|---|---------|------|--------------|
| 1 | 00-sources/index.md | 45 | 入口索引文件 |
| 2 | 00-sources/ch01-intro.md | 320 | 引言，核心声称 |

## 结构索引

[跨所有 .md 文件扫描标题/表格/公式/代码块，标注 文件名:行号]

- 章节标题：搜索 "^#+ "
- 数学公式块：搜索 "$$" 或 "\\["
- 表格：搜索 "^|" 或 "表\\d"
- 代码块：搜索 "```"

## 语义线索（01-dissect-concept 后追加）

<概念层解剖完成后追加，行号引用格式为 文件名:行号>
```

### D5：引用格式统一

| 模式 | 引用格式 | 示例 |
|------|---------|------|
| 单文件（粘贴/快照/引用） | `第<N>行："直接引用原句"` | `第42行："Scaling Law 已经触顶"` |
| 目录模式 | `文件名:<N>："直接引用原句"` | `ch02-method.md:42："Scaling Law 已经触顶"` |
| 连续行范围 | `第<N>-<M>行` 或 `文件名:<N>-<M>："..."` | `第42-45行` 或 `ch02.md:42-45："..."` |

**格式规范：**
- 行号前统一使用半角冒号 `:`
- 引用内容统一使用中文引号 `""`
- 范围用半角连字符 `-`

### D6：00-todolist.md 原文索引格式（支持五种输入形态）

```markdown
## 原文索引

- 输入模式：[粘贴 / 单文件快照 / 单文件引用 / 目录快照 / 目录引用]
- 源路径：<原始文件或目录的绝对路径>                ← 快照/引用模式时有值
- 单文件路径：<00-original.md 的绝对路径>            ← 粘贴/单文件快照时有值
- 或引用源文件：<源文件的绝对路径>                   ← 单文件引用时有值
- 源目录路径：<00-sources/ 的绝对路径>               ← 目录快照时有值
- 或引用源目录：<源目录的绝对路径>                    ← 目录引用时有值
- 索引入口：<00-index.md 的绝对路径>                 ← 目录模式时有值
- 外部索引：<用户提供的 index 文件路径>               ← 用户提供时有值，否则为"无"
- 存储模式：[快照 / 引用]
- 总行数：<自动统计>
- 文件 hash（引用模式）：<启动时各源文件的 MD5>      ← 引用模式时记录
```

### D10：aolun-arming 路由规则修改

当前 aolun-arming 的路由逻辑存在断裂：按输入字符数 ≥ 1500 路由到 fileflow，但文件路径字符串远少于 1500 字符，导致路径输入永远不触发 fileflow。

**修改 arming 的前置判断流程：**

```
| 条件 | 路径 |
|------|------|
| 输入是文件路径（以 / ~ ./ 开头，或匹配路径模式） | → 移交给 aolun-fileflow |
| 输入是目录路径 | → 移交给 aolun-fileflow |
| 输入文本字符数 ≥ 1500 | → 移交给 aolun-fileflow |
| 输入文本字符数 < 1500 | → 进入意图问询 |
```

**路径判断规则：**
- 以 `/` 开头：绝对路径
- 以 `~` 开头：家目录路径
- 以 `./` 或 `../` 开头：相对路径
- 包含文件扩展名（`.md`、`.txt` 等）且无连续段落文本特征
- 以路径分隔符结尾（目录标识）

**修改范围：** `skills/aolun-arming/SKILL.md` 的 `## 前置：输入长度判断` 小节。

**不再列为"不改动的文件"。**

### D11：搜索策略自适应

当前 `00-todolist.md` 的"搜索策略"部分包含硬编码的 LLM 相关搜索关键词。在目录模式或非 LLM 领域的场景下，这些关键词完全不适用。

**修改方案：**

1. fileflow Part 1 生成 `00-todolist.md` 时，将搜索策略改为**根据原文内容动态生成**：

```
搜索策略生成规则：
- 从 00-original.md 或 00-index.md 的结构索引中提取高频术语
- 生成 3-5 条领域相关的通用搜索策略
- 保留以下通用策略（对所有领域适用）：
  - 核心声称：搜索"提出"、"认为"、"表明"、"证明"
  - 数据和证据：搜索数字、百分比、引用来源
  - 定义关键术语：搜索"所谓"、"定义为"、"是指"
- 01-dissect-concept 完成后，语义线索替代/补充这些通用策略
```

2. 搜索策略不再硬编码在 `00-todolist.md` 格式模板中，而是作为动态生成逻辑写入 fileflow Part 1 的流程说明。
