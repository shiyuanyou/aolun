# 利益层解剖器

> "你要先问：这篇文章是谁写的？他为什么要写？他得到什么好处？
>  问清楚这三个问题，再来看他说了什么。"
> —— 李敖

## 核心任务

**找到信息后面的人，和他们的动机。** 没有立场中立的技术论断。利益层解剖不是阴谋论——它是把隐藏的利益关系显现出来，让读者知道他们拿到的是什么质地的信息。

---

## 解剖步骤

### 第一步：识别信息源的身份与立场

**作者身份分析：**
- 作者/机构是谁？（供应商、咨询公司、学术界、独立工程师、媒体）
- 他们与这项技术有什么利益关系？（是否是技术的创造者/销售者/投资方）
- 他们的历史立场是什么？（一贯推某类技术的人？平台方？）

**利益关系矩阵：**

| 身份 | 典型利益 | 典型偏差方向 |
|------|---------|------------|
| 供应商/厂商 | 卖产品/服务 | 放大优点，隐藏约束 |
| 咨询公司 | 卖实施服务 | 推复杂方案，扩大项目范围 |
| 投资方/VC | 退出回报 | 夸大市场规模，制造紧迫感 |
| 平台方 | 生态锁定 | 推生态内方案，贬低外部替代 |
| 媒体/KOL | 流量/影响力 | 追热点，放大新奇性 |
| 学术界 | 发表/影响力 | 过度解读实验结果的适用范围 |
| 独立工程师 | 通常偏中立 | 可能有技术偏好盲区 |

### 第二步：分析信息选择的模式

信息是被选择过的。检验选择性：

**选择性呈现检验：**
- 只展示成功案例，没有失败案例？
- 只引用支持立场的数据，过滤掉反例？
- 对比对象是经过挑选的稻草人（最差的替代方案）？
- 时间范围是否被刻意截断（只选对自己有利的时间段）？

**信息密度不均检验：**
- 优点部分：详细、具体、有数据
- 缺点部分：模糊、笼统、"在某些情况下"
- 约束部分：完全缺席或一句带过

### 第三步：追溯利益链

从直接利益推手往后追：

```
直接推手（作者/公司）
    → 谁资助了这篇文章/研究？
    → 谁会因为这项技术被采用而获利？
    → 谁在这项技术的失败中受损？
```

**利益链的典型形态：**
- **供应商资助的"独立研究"**——结论往往有预设
- **平台生态推广**——以开源姿态掩盖生态锁定意图
- **行业协会标准推广**——大公司通过标准制定锁定竞争优势
- **咨询公司最佳实践**——制造对咨询服务的依赖

### 第四步：评估利益偏差的影响程度

| 偏差程度 | 判断标准 | 处理建议 |
|---------|---------|---------|
| 轻度 | 有利益关系但信息基本准确，只是选择性强调 | 补充对立信息后参考 |
| 中度 | 关键约束被系统性隐藏 | 需要独立验证核心主张 |
| 重度 | 存在明显虚假信息或数据造假 | 此信息源不可信，需要全面独立核实 |

```
  - 历史立场：

信息选择性分析：
  - 被放大的内容：
  - 被隐藏的内容：
  - 对比对象的选取是否公平：

利益链追溯：
  - 直接获益方：
  - 间接获益方：
  - 若失败受损方：

偏差程度评估：[轻度 / 中度 / 重度]

利益层综合判断：
[这篇帖子代表谁的立场，读者应该以什么心态使用这份信息]
```
