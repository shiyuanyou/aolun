# 攻击文生成器

> "我写文章，不是为了骂人。骂人是小事。我是要让真相出来，让读者看见这件事情的真面目。
>  骂，只是让读者在看清真相之后，知道他们有多荒唐。"
> —— 李敖

## 核心任务

**把解剖和扫描的结论，写成一篇有力量、有品质、可以发布的战斗文本。**

这篇文章要做三件事：
1. **拆穿**——把目标的弱点展示给读者，让读者自己得出结论
2. **重建**——指出更好的方向，而不只是破坏
3. **风格**——李敖的腔调，使结论有记忆点

**写作流程总览：**

```
第一层：辩证诊断  →  第二层：攻击执行  →  第三层：实践检验  →  第四层：认识的螺旋
（打哪里）            （怎么打）            （打完查漏）         （发布决策）
```

## 第一层：辩证诊断

> 来源：《矛盾论》（毛泽东，1937）
> 目的：开打前把目标的内部矛盾结构摸透。不做诊断，就不知道把武器对准哪里。

### 1.1 主要矛盾定位

目标内部有多个矛盾，必须找到主要矛盾——"其中必有一种是主要的，起着领导的、决定的作用"。

**操作：**
- 列出目标中存在的所有内在张力（3–6条，列出后合并同类项，保留本质不同的张力）
- 判断哪一条"规定或影响着其他矛盾的存在和发展"
- 一句话陈述主要矛盾：`[A]与[B]之间的矛盾，其中[A/B]处于支配地位`

**禁止跳过：** 如果只看到一个矛盾，说明分析深度不够。任何论证内部都有多个张力，主要矛盾是从多个张力中选出来的，不是凭直觉第一眼看到的那个。

### 1.2 矛盾特殊性分析

"不同质的矛盾，只有用不同质的方法才能解决。"不同类型的矛盾对应不同的解构路线。

**矛盾类型分类（四种）：**

| 类型 | 特征 | 对应主攻路线 |
|------|------|------------|
| 概念混乱型 | 关键词被偷换或混用，同一词在不同语境意思不同 | 暴露定义跳跃，精确指出在哪句话里偷换 |
| 机制断裂型 | 因果链中间某环节缺失或虚构，结论无法从前提推出 | 还原隐藏约束，证明被省略的前提不成立 |
| 利益遮蔽型 | 论证服务于未申明的利益，客观外衣下藏着立场 | 以子之矛攻子之盾，用对方自己的说法推翻对方 |
| 历史错位型 | 把历史上已经失败的方案重新包装，换了外壳本质未变 | 引出荒谬结论，让历史重演的可笑自我显现 |

**操作：**
- 判断主要矛盾属于哪种类型
- 选定一条主攻路线（其他三种作为补充验证，不作为主路线）
- 一个目标可以同时属于多种类型，但**主路线只有一条**

### 1.3 主要方面 vs 次要方面

"事物的性质，主要地是由取得支配地位的矛盾的主要方面所规定的。"

**操作：**
- 确认主要矛盾的两个方面各是什么（不是两个人，是两种力量或两种逻辑）
- 判断哪个方面目前处于支配地位（即对方论证依赖哪个方面来成立）
- 攻击策略：集中打击支配地位的那个方面，使其在读者眼中转化为次要方面

**注意：** 矛盾的主次方面是可以转化的——"矛盾的主要和非主要的方面互相转化着，事物的性质也就随着起变化"。攻击文的目标，就是推动这个转化在读者认知中发生。

### 1.4 同一性分析：转化条件

"矛盾着的双方，依据一定的条件，各向着其相反的方面转化。"

**操作：**
- 找出主要矛盾发生转化需要什么条件
- 这个条件就是"立"的方向——不是外加建议，而是目标内部矛盾运动的自然出口
- 一句话陈述转化条件：`当[条件X]成立时，[A]将让位于[B]`

**关键：** "立"不是批评者的发明，而是从被批评对象自身的矛盾运动中发现的。这是"破而后立"和"只破不立"的根本区别。

### 1.5 对抗性判断

"对抗是矛盾斗争的一种形式，而不是矛盾斗争的一切形式。"

**操作：** 判断这个矛盾是否具有对抗性。

- **对抗性矛盾**（利益根本对立，无法调和，改变对方不在目标之列）：攻击要彻底，反讽要锋利，不留余地，目的是让读者看清而非说服对方
- **非对抗性矛盾**（认知错误或利益可调和，改变对方是有可能的）：攻击要精准，建构方向要具体，给对方留改正空间，反讽用同情式而非鄙视式

### 输出：矛盾定位（混合模式：按文体决定是否显示）

完成以上五步后，输出 **2-3 句矛盾定位**：

> **矛盾定位：**
> [主要矛盾一句话，格式：A与B之间的矛盾，其中X处于支配地位]
> [矛盾类型 + 主攻路线一句话]
> [转化条件/立的方向一句话]

**呈现规则：**
- 标准攻击文、深度拆解：矛盾定位出现在正文之前，作为写作框架的显式声明，**不计入正文字数**
- 快速评论：矛盾定位**内部使用，不输出到正文**

## 第二层：攻击执行

> 三件武器保留，但每件武器的选择标准和执行路线由第一层的诊断结果决定，不再套模板。

### 武器一：征引

李敖攻击，必先列证据。不是"据说""大家都知道"，而是**具体的**——原文在哪里、数据是什么、历史上的先例是什么。

**升级驱动：** 征引的选择不再是"找到一条证据"，而是"找到能暴露主要矛盾的证据"。

根据 1.2 的矛盾类型选择征引目标：

- **概念混乱型**：引用对方在不同语境下对同一概念的不同用法（让偷换在原文中显形）
- **机制断裂型**：引用对方断言因果关系但省略中间环节的原文（让断裂在原文中显形）
- **利益遮蔽型**：引用对方既声称客观又暴露立场的原文（让利益在原文中显形）
- **历史错位型**：引用对方的核心主张 + 历史上相同主张的原始来源（让重演在对比中显形）

**征引的节奏：**
- 先引用对方的说法（原文或原意）
- 然后摆出与之矛盾的事实或逻辑
- 让对比说话，不要过早下判断

**征引的要求：**
- 每一个主要指控，至少一条具体依据
- 依据来自解剖和扫描的结论，不是凭空想象
- 对方原话优先——让对方的话变成攻击对方的武器

### 武器二：解构

征引建立了战场，解构是肉搏。把对方论证的骨架拆出来，逐节折断。

**四种技法：**

1. **以子之矛攻子之盾**：用对方文中的一个说法，推翻对方的另一个说法（对应利益遮蔽型）
2. **暴露定义跳跃**：找出概念偷换的那一刻，精确指出在哪句话里（对应概念混乱型）
3. **还原隐藏约束**：把对方没说的前提条件摆出来，然后证明这个前提不成立（对应机制断裂型）
4. **引出荒谬结论**：把对方的逻辑推到极端，让结论的荒谬自我显现（对应历史错位型）

**升级驱动：** 由 1.2 的矛盾类型选定主路线，其余三种作为补充验证。

**解构的节奏：**
- 从最小的、最确定无疑的漏洞开始（让读者先认同你的判断力）
- 逐步升级到最核心的弱点
- 不要一开始就扔最大的炸弹，让批评的力度逐渐积累

**新增终点约束：** 解构的终点不是"证明对方错了"，而是"暴露内部矛盾，让转化条件（1.4）显现"。解构结束的标志是：读者已经能看到，在什么条件下对方的主要方面会让位于次要方面。

### 武器三：反讽

征引和解构之后，读者已经看清楚了。反讽是最后一刀——在逻辑胜利之后，给对方一个优雅的定性。

**反讽的条件：**
- 只有在证据和逻辑已经充分之后，才使用反讽
- 反讽的对象是论断和行为，不是人格
- 好的反讽是让读者会心一笑，不是让读者感到恶意

**升级驱动：** 反讽的落点由 1.3 决定——反讽的是目前处于支配地位的那个方面，而不是对方整体。

根据 1.5 的对抗性调整力度：
- **对抗性矛盾**：反讽可以更锋利，目标是让对方的支配地位在读者眼中彻底崩塌
- **非对抗性矛盾**：用"同情式挖苦"，指出荒谬但不封死改正空间

**李敖式反讽的几种形态：**
- **过度承认**（适用于机制断裂型、利益遮蔽型）：承认对方某个小优点，但用这个承认衬托大缺陷（"不是没有道理，只是这点道理，不值得付这么大的代价"）
- **同情式挖苦**（适用于概念混乱型、非对抗性矛盾）：以理解的姿态揭示荒谬（"我能理解为什么会这么想，但这个想法的代价，他们大概还没算清楚"）
- **历史比较**（适用于历史错位型）：用已征引确立的历史对比给出最终定性句——不是再次引新史料，而是对已有证据的情绪收尾（"这个想法在1970年代有个名字，叫……"——LLM 根据目标主题补全历史名称）

## 第三层：实践检验

> 来源：《实践论》（毛泽东，1937）
> 目的：文章写完后，用两条轨道检验质量。两轨独立执行，互不影响。

### 轨道 A：建构落地性检验

> "不在于懂得了客观世界的规律性，因而能够解释世界，而在于拿了这种对于客观规律性的认识去能动地改造世界。"

文章里的"立"——建构方向——必须同时满足三条：

**① 有实践来源**
这个方向来自哪个领域的实践经验？不能是纯理论推导。
要能回答：这个建议在哪里被验证过？谁用过？结果如何？

**② 有边界条件**
在什么情况下这个方向成立？在什么情况下不成立？
没有边界条件的建议是空话——任何建议都有它失效的条件。

**③ 有可操作性**
读者读完之后，能做的第一个具体动作是什么？
检验方法：把"立"的部分单独提取出来，问：如果对方想按这个方向走，第一步做什么？如果答不上来，"立"就还是空话，必须重写。

### 轨道 B：对手反击模拟

> "实践、认识、再实践、再认识，这种形式，循环往复以至无穷。"

文章写完后，模拟对手最可能的三种反击，并在文章中预埋伏笔：

**① 最强反击**
如果对手的论证中有一个我没有充分处理的方面，是哪个？
处理方式：在文章中预埋一句话，承认这个方面存在，但说明为什么它不影响主要结论。

**② 转移反击**
对手最可能把话题引向哪个方向来回避主要矛盾？
处理方式：在文章中点名这个转移路径，让读者预先识别它。

**③ 情绪反击**
对手最可能用什么情绪化的说法来绕过逻辑？（如："你这是攻击个人""你不懂这个领域"）
处理方式：在文章的语言选择上预先排除这类指控的成立条件（只批评论断和行为，不批评人格）。

对每种反击，给出一句预备回应（不必展开，只需在文章中预埋伏笔）。

## 第四层：认识的螺旋

> 来源：《实践论》
> 核心：不完美中前行。

《实践论》明确反对两种错误：
- **右倾机会主义**："思想不能随变化了的客观情况而前进"——等到完美再发布，就是这个
- **左翼空谈主义**："思想超过客观过程的一定发展阶段"——过度引申、言过其实，也是错的

### 够用就发，不等完美

每篇攻击文都是当前认识阶段的产物。判断标准不是"有没有漏洞"（一定有），而是三条：

- 主要矛盾是否被准确定位？
- 最核心的弱点是否有充分依据？
- "立"的方向是否可操作？

**三条满足，就可以发布。** 不满足的部分，留给下一轮。

发布不是认识的终点，是认识进入下一轮实践的起点。

### 发布即实践：对手反应是材料

发布之后，对手的反击、读者的疑问、新的事实出现，都是"再实践"的材料。

**下一轮写作时，用这些材料更新认识：**
- 记录对手的实际反击（对比轨道B的预测，看预测是否准确）
- 如果出现没有预测到的反击，把它加入下一轮第一层诊断的矛盾清单（1.1）
- 如果"立"的方向被证明不可操作，在下一轮更新转化条件（1.4）
- 如果主要矛盾的判断被证明错误，重新做诊断，从头来

### skill 本身也在迭代

这个 skill 是当前认识阶段的产物。每次使用后，如果发现某个步骤不够用或者产生了误导，应该更新 skill 本身。

**这个 skill 不是终点，是起点。**

## 文章结构模板

### 结构一：标准攻击文（适合技术评论）

```
矛盾定位：[2-3句，写作框架声明，不计入正文字数]

第一节：问题的提出（50-100字）
  [引用原帖的核心主张，不加评论，让读者自己先读到]
  [提出一个悬念：这个主张是否真的成立？]

第二节：概念解剖（200-400字）
  [这个东西真正是什么]
  [定义里藏了什么]
  [概念和历史上的先辈们]

第三节：机制与约束拆解（300-500字）
  [因果链的哪里断了]
  [真实条件下会发生什么]
  [被刻意隐藏的边界条件]

第四节：利益与动机（100-200字）
  [谁在推这个，他们得什么好处]
  [信息选择背后的议程]

第五节：他山之石（200-400字）
  [真正的问题是什么]
  [其他领域是怎么解决这个问题的]
  [更好的方向在哪里——即转化条件（1.4）的展开]

第六节：结语（50-150字）
  [不是总结，是定性]
  [反讽的刀，一刀收尾]
```

### 结构二：快速评论（适合推文/短帖回复）

```
矛盾定位：[内部使用，不输出到正文]

第一句：复述对方核心主张（5-20字）
中间：一个最致命的弱点 + 一条具体证据（50-150字）
最后：一句有力量的定性（10-30字）
```

### 结构三：深度技术拆解（适合长文）

```
矛盾定位：[2-3句，作为引言的前置框架]

引言：这篇文章要做什么
第一部分：解剖（四层全写）
第二部分：弱点清单（逐条列出，附依据）
第三部分：历史上的前辈（案例 + 结局）
第四部分：谁在推、为什么推
第五部分：更好的解法（跨领域方案，即转化条件的展开）
结论：这个东西的真实定位是什么，应该在什么场合用、什么场合不用
```

## 语言风格规范

### 李敖语言的核心特征

**直接**：不用被动语态掩盖判断，不用"可能""也许"软化结论
**精确**：用数字、时间、名称，不用"很多""通常""大部分"
**有节奏**：长句和短句交替，论证密集处用长句，结论处用短句
**文白相间**（如适用）：偶尔用文言词汇或历史典故，增加厚重感
**自信**：论断直接给出，不先铺垫"我个人认为"

### 语言禁区

- 不用"我觉得""我认为"（用事实和逻辑代替）
- 不用"这是一个复杂的问题"（这是回避的遁词）
- 不用"每种方法都有其适用场景"（这是没有立场的伪中立）
- 不用"当然，也有一些优点"（除非真的要用这个优点来衬托缺陷）
- 不堆砌形容词（"革命性""颠覆性"是对方的用法，不是批评者的用法）

### 结构标记

在文章关键转折处使用明确的逻辑信号词：
- 征引："原文写道……""按照他的说法……"
- 解构："问题在于……""这个推论的跳跃发生在……"
- 反讽："不得不承认……（但）""这个想法的历史先辈……"
- 建构："真正的问题是……""从[X领域]的角度看……"

## 质量自检

发布前过以下检查：

**【第一层：辩证诊断】**
- [ ] 主要矛盾已定位，不是表面现象（至少从3条张力中选出）
- [ ] 矛盾类型已确认，主攻路线已选定
- [ ] 主要方面已识别，攻击对准支配地位的那个方面
- [ ] 转化条件已找到，"立"从内部矛盾中生长而非外加
- [ ] 对抗性已判断，反讽力度与之匹配

**【第二层：攻击执行】**
- [ ] 每个主要指控有具体依据（来自解剖/扫描结论）
- [ ] 征引选择与矛盾类型匹配
- [ ] 解构路线对应主攻路线，不是套模板
- [ ] 解构终点已到达（读者能看到转化条件）
- [ ] 反讽在论证之后，落点在主要方面
- [ ] 没有用气势代替论证

**【第三层：实践检验】**
- [ ] "立"有实践来源，有边界条件，有可操作性
- [ ] 能回答：读者第一个具体动作是什么
- [ ] 三种对手反击已模拟，伏笔已预埋
- [ ] 没有人身攻击，批评的是论断和行为

**【认识的螺旋：发布决策】**
- [ ] 三条发布标准满足（主要矛盾定位 ✓、核心弱点有据 ✓、立可操作 ✓）
- [ ] 对方优点（如果真有）被公平提及
- [ ] 字数与信息密度匹配，没有冗余铺垫
