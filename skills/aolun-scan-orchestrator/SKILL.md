# 扫描层并行编排器

> "兵贵神速，更贵合围。四面同时出击，对手连逃跑的方向都找不到。"
> —— 李敖

## 核心职责

接收解剖层输出摘要，以**单次消息**并行 dispatch 四个扫描器 subagent，汇总返回结果为统一的扫描综合报告。

**关键原则：**
- 四个 subagent 必须在同一条消息中发出（真正并行）
- subagent 通过文件路径+ 搜索指令获取原文和上游输出，**不通过 prompt 粘贴文本内容**
- 综合报告是唯一传递给下游的内容，四份 subagent 全文报告不进入主窗口

---

## 输入来源判断

本 skill 有两种运行模式，取决于上游是否通过 fileflow 派发：

| 运行模式 | 判断条件 | 原文获取方式 | 上游输出获取方式 |
|---------|---------|------------|-------------|
| fileflow 模式 | 存在 `00-todolist.md` 和 `00-original.md` | 读取文件路径，subagent 自行搜索 | 读取文件路径，subagent 自行读取 |
| 内存模式（短文本） | 不存在上述文件 | 完整原文已在主 agent 上下文中 | 解剖摘要已在主 agent 上下文中 |

**fileflow 模式优先。** 如果检测到 fileflow 任务目录（通过 `00-todolist.md` 存在性判断），必须使用 fileflow 模式。

---

## 执行步骤

### Step 1：准备解剖层输入摘要

从解剖层四份报告中提炼以下格式（约 200 字）：

**fileflow 模式：** 从 `00-todolist.md` 状态表的摘要列和各步骤输出文件的 `【摘要】` 块中提取。

**内存模式：** 从主 agent 上下文中的解剖层输出中提炼。

摘要格式：
```
【解剖层输入摘要】
概念层：[核心声称] + [最关键的概念漏洞 1-2条]
机制层：[声称的原理] + [因果链中最脆弱的环节]
约束层：[被隐藏的关键约束] + [边界外失效场景]
利益层：[主要推动方] + [信息选择偏差方向]
```

### Step 2：单次消息并行 dispatch 四个 subagent

**在同一条消息中发出四个 Task 工具调用**（不得分开发送）：

> **平台注：** 在不支持 Task 并行调用的宿主中（如 Cursor、Codex），可改为依序执行四个扫描器，汇总方式相同，只是无法并行加速。

---

### fileflow 模式下的 dispatch 模板

每个 subagent 的 prompt 统一使用以下结构（只传文件路径和搜索策略，不传文本内容）：

```
你是 <扫描器名称>。使用 aolun-<扫描器> skill 对目标文本进行扫描。

【文件路径】
- 原始文件：<00-original.md 的完整路径>
- 索引与状态：<00-todolist.md 的完整路径>
- 上游输出文件：
  - 概念层：<01-dissect-concept.md 的完整路径>
  - 机制层：<02-dissect-mechanism.md 的完整路径>
  - 约束层：<03-dissect-constraint.md 的完整路径>
  - 利益层：<04-dissect-interest.md 的完整路径>

【执行指令】
1. 先读 00-todolist.md 中的"原文索引"和"搜索策略"部分，获取搜索线索
2. 使用 Read 和 Grep 工具直接搜索原始文件，找到关键证据
   - 先读 00-original.md 建立全局理解（可分段读取）
   - 根据你的扫描职责，重点搜索对应类型的证据
   - 每个弱点必须附有原文行号和直接引用
3. 如需参考上游分析结论，Read 对应的上游输出文件
4. 按照 aolun-<扫描器> 的 SKILL.md 指令执行完整扫描

【引用硬约束】
- 每个弱点指控必须附原文行号 + 直接引用（连续引用不超过3行，超过用行号范围标注）
- 禁止转述代替引用——转述必然失真，引用才是弹药
- 搜索未果时必须声明"未找到直接证据"，不许脑补

【输出格式】
在扫描报告末尾追加以下摘要块：

【致攻击层的摘要】（来自 <扫描器>）
最致命的一击：[一句话]
弱点清单（最多5条，按攻击力排序）：
  1. [类型] — [原文证据，直接引用原句] — 攻击力：[高/中/低]
  2. ...
```

四个扫描器各自的差异：

**Task 1 → scan-logic** 重点搜索指令：
```
- 搜索核心论点的因果连接词（因此、所以、这意味着、表明、证明了）
- 搜索同一概念在不同章节的定义，寻找内部矛盾
- 搜索所有数字声称，验证引用来源
- 搜索类比论证段落，检查类比是否成立
```

**Task 2 → scan-engineering** 重点搜索指令：
```
- 搜索技术参数、性能指标、延迟数据
- 搜索系统架构描述和依赖声明
- 搜索实验设置（数据集规模、硬件环境、基线版本）
- 搜索消融实验，检查是否有替代解释
```

**Task 3 → scan-history** 重点搜索指令：
```
- 搜索相关工作、文献综述段落
- 搜索"首次"、"首个"、"novel"、"first"等创新性声称
- 搜索方法与先前工作对比的段落
- 搜索历史上同类方案的成功/失败记录
```

**Task 4 → scan-motive** 重点搜索指令：
```
- 搜索作者立场声明、研究动机段落
- 搜索利益相关方描述、资助信息
- 搜索强调词（显著、极大、关键、重要）密集段落
- 搜索对比基准选择，检查是否有回避
```

### 内存模式（短文本）下的 dispatch 模板

当没有 fileflow 任务目录时，原文和上游输出在主 agent 上下文中。

**原文传递规则（dispatch 前必读）：**

> 原文长度决定传递方式，两种情况均不得对原文做语义压缩或改写：
>
> - **≤ 8000 token**：将原文完整复制进每个 subagent 的 `[原文]` 字段，一字不改。
> - **> 8000 token**：不应使用内存模式，应切换到 fileflow 模式。

每个 subagent 的 prompt 结构：

```
你是一个<扫描器名称>。使用 aolun-<扫描器> skill。

【输入】
解剖层摘要：
[粘贴解剖层输入摘要]

原文：
[粘贴原文——完整原文，未经改写]

【引用约束】
弱点清单中的"原文证据"字段必须从上方原文中直接复制原句，用引号标注，不得意译或概括。

【要求】
执行 aolun-<扫描器> 的完整扫描流程，然后在报告末尾追加以下摘要块：

【致攻击层的摘要】（来自 <扫描器>）
最致命的一击：[一句话]
弱点清单（最多5条，按攻击力排序）：
  1. [类型] — [原文证据，直接引用原句] — 攻击力：[高/中/低]
  2. ...
```

各扫描器的具体扫描重点同 fileflow 模式中的重点搜索指令。

---

### Step 3：汇总综合报告

四个 subagent 全部返回后，从每份报告中提取 `【致攻击层的摘要】` 块，汇总为：

```
【扫描综合报告】
（由 aolun-scan-orchestrator 汇总，共4个维度）

逻辑维度最致命：[来自 scan-logic 的"最致命的一击"]
工程维度最致命：[来自 scan-engineering 的"最致命的一击"]
历史维度最致命：[来自 scan-history 的"最致命的一击"]
动机维度最致命：[来自 scan-motive 的"最致命的一击"]

全弱点清单（攻击力排序，去重合并，最多10条）：
  1. [弱点类型] — [维度] — [原文证据] — 攻击力：[高/中/低]
  2. [弱点类型] — [维度] — [原文证据] — 攻击力：[高/中/低]
  3. ...
```

**汇总规则：**
- 按攻击力（高→中→低）排序
- 同一证据被多个扫描器引用时合并为一条，注明多维度
- 超过10条时保留攻击力最高的10条

**fileflow 模式下：** 将综合报告写入 `05-scan-summary.md`，并在文件顶部包含 `【摘要】` 块。

---

## 向下游传递

**fileflow 模式：** 下游 skill 通过 `00-todolist.md` 获取综合报告路径。

**内存模式：** 综合报告直接传递给：
- `aolun-other-mountains`：工程弱点清单作为跨领域解法的需求入口
- `aolun-attack`：完整综合报告作为攻击文弹药库
